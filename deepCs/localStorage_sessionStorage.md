# 📂 브라우저 저장소 비교: Cookie vs Web Storage

브라우저 저장소는 크게 **쿠키**와 **웹 스토리지**로 나뉘며, 웹 스토리지는 다시 **로컬 스토리지**와 **세션 스토리지**로 구분됩니다.

---

## 1. 한눈에 비교하는 요약 테이블

| 구분 | 쿠키 (Cookie) | 로컬 스토리지 (Local Storage) | 세션 스토리지 (Session Storage) |
| :--- | :--- | :--- | :--- |
| **저장 위치** | 브라우저 & 서버 전송 | 브라우저 (클라이언트) | 브라우저 (클라이언트) |
| **데이터 유효 기간** | 설정 시까지 (영속) / 종료 시까지 (세션) | 삭제하지 않는 한 영구적 | **탭/윈도우를 닫을 때** 삭제 |
| **저장 용량** | 매우 작음 (약 4KB) | 큼 (약 5MB ~ 10MB) | 큼 (약 5MB ~ 10MB) |
| **서버 전송 여부** | **모든 HTTP 요청 시 자동 전송** | 전송되지 않음 | 전송되지 않음 |
| **주요 용도** | 로그인 세션 관리, 광고 트래킹 | 자동 로그인, 사용자 설정 저장 | 입력 폼 데이터, 일회성 인증 |

---

## 2. 주요 저장소별 상세 특징

### 🍪 쿠키 (Cookie)
서버와 클라이언트 간의 **상태를 유지**하기 위해 고안된 전통적인 방식입니다.
- **유형:** - **세션 쿠키:** 만료 시간을 정하지 않아 브라우저 종료 시 삭제됩니다.
  - **영속 쿠키:** `Max-Age`나 `Expires`를 설정해 디스크에 저장되며 재방문 시에도 유지됩니다.
- **특징:** 매번 서버로 전송되므로 네트워크 오버헤드가 발생할 수 있습니다.
- **보안 옵션:**
  - `HttpOnly`: 자바스크립트로 접근 불가 (XSS 방어).
  - `Secure`: HTTPS 프로토콜에서만 전송.
  - `SameSite`: CSRF 공격 방어 설정.

### 💾 로컬 스토리지 (Local Storage)
사용자가 직접 삭제하거나 브라우저 캐시를 비우지 않는 한 데이터가 영구적으로 남아 있습니다.
- **특징:** 도메인마다 별도의 저장 공간을 가지며 용량이 넉넉합니다.
- **용도:** 자동 로그인 여부, 다크모드/라이트모드 설정, 비로그인 장바구니 등.
- **주의:** 자바스크립트로 쉽게 읽을 수 있어 민감 정보 저장 시 **XSS 공격**에 취약합니다.

### ⏱️ 세션 스토리지 (Session Storage)
현재 떠 있는 **탭(Tab) 또는 윈도우 내에서만** 유효한 저장소입니다.
- **특징:** 페이지를 새로고침해도 데이터는 유지되지만, 탭을 아예 닫으면 바로 소멸합니다.
- **용도:** 일회성 로그인 정보, 입력 중인 폼 데이터, 상세 페이지 진입 전 스크롤 위치 등.

---

## 3. 결론: 언제 무엇을 쓰는가?

- **쿠키:** 서버와 직접 소통해야 하는 **인증(세션 아이디, 토큰)** 및 광고 트래킹.
- **웹 스토리지:** 서버가 굳이 실시간으로 알 필요가 없는 **UI/사용자 편의 데이터** 및 오버헤드를 줄여야 하는 대용량 클라이언트 데이터.
